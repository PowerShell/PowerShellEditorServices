API Reference

The .NET API for PowerShell Editor Services is organized in a way that allows you to easily get started using all of its services but also giving you the option to only use the services you care about in your application.

The best starting point is the EditorSession class which can start up all of the following services for use in a single editing session.

Use the LanguageService to provide language intelligence behaviors like finding the references or definition of a cmdlet or variable.

Use the AnalysisService to provide rule-based analysis of scripts using PowerShell Script Analyzer.

Use the DebugService to easily interact with the PowerShell debugger.

Use the ConsoleService to provide interactive console support in the user's editor.

Use the ExtensionService to allow the user to extend the host editor with new capabilities using PowerShell code.

The core of all the services is the PowerShellContext class. This class manages a session's runspace and handles script and command execution no matter what state the runspace is in.
